什么时候读：每次启动新赛道或新视频创作前；每次讨论“这条内容到底在做什么”时。
是不是唯一真源：是；关于“猫图腾上游剧本思维框架”的定义以本文档为准。

# 07 人的载体型剧本思维框架

## 核心摘要
- 猫图腾不是“漫改风格的猫IP（人格化内容形象）”，而是“人的心理状态的载体”。
- 视觉上是猫，内核上是人；猫的任务不是表演动作，而是外化一种现代人的具体心理状态。
- 剧本起点不再是“猫要做什么”，而是“目标用户此刻正处于什么心理状态”。
- 主要评估标尺从“画面精细度”切换为“投射密度”；画面质量不被放弃，但降级为辅助标尺。
- 本文档与 `06_宏观框架先行原则.md` 并列，和规划卡v1.0同属每次启动 sprint 前必读文件。

## 零｜为什么需要这个文档

本文档诞生于赛道1第一条视频迭代过程。

在完成“前中后景首尾帧 → SD2直出 → Agent模式逐帧修复”后，识别出被长期忽视的根本问题：问题不在 prompt、不在画质，也不在单帧是否更精细，而在“猫图腾到底是什么”没有共识。

其中一个关键触发点是：闭笔记本这类动作，本质上更像真人情绪表达中的填补式动作，而不是猫天然会做、且能稳定成立的动作。它可以在抖音语境里触发即时理解，但如果没有上游内核支撑，就会变成“看起来像有情绪，实则没有投射靶心”的空动作。

对这个问题的回答是：猫图腾是“人的载体”，不是“漫改风格的猫IP（人格化内容形象）”。

本文档把这个定位转化为可执行的剧本思维框架，让所有下游动作从它自然推导，不再在每一轮创作里重复做战略判断。

## 一｜核心命题

### 1.1 定位命题

视觉上是猫，内核上是人。

- 视觉：猫（银白色 + 淡紫灰耳尖 / 尾尖，极简扁平插画风格）
- 内核：现代人的某种具体心理状态
- 错位结构：猫的形象 + 人的内心 = 投射载体

### 1.2 区分命题

| 类型 | 典型代表 | 观众与猫的关系 |
|---|---|---|
| 真猫博主 | 橘猫日常、英短喜剧 | “它是只可爱的动物” |
| 拟人化猫 | 汤姆猫、加菲猫 | “看猫装成人多好笑” |
| 虚拟萌宠IP | 各种卡通形象 | “它像一个角色” |
| 猫图腾 | — | “它就是我” |

### 1.3 价值主张

猫图腾为现代人（尤其一二线城市18–28岁独居女性）提供情绪投射载体——让目标用户在0.5秒内把自己投射进去，并通过“这是只猫”的距离感，获得“这就是我”的认同感。

## 二｜人的载体型剧本框架

### 2.1 核心框架

创作起点从“猫要做什么动作”，转变为“目标用户的什么心理状态要被外化”。

默认四步结构：

开头 = 观众的某种心理状态  
中段 = 该状态的内部变化  
结尾 = 该状态的某个凝固瞬间  
动作 = 心理状态的外化载体

### 2.2 与传统剧本思维的区别

| 维度 | 传统剧本思维 | 人的载体型剧本思维 |
|---|---|---|
| 起点 | 猫在做什么 | 观众此刻的心理状态 |
| 推进逻辑 | 动作 → 动作 → 动作 | 状态 → 内部变化 → 凝固瞬间 |
| 猫的角色 | 主角 | 心理状态的外化载体 |
| 评估问题 | 动作是否流畅 | 心理状态是否被准确外化 |

### 2.3 使用示例

- 错误起点（动作层）：让猫扭头看窗外，然后回头，最后盖上笔记本。
- 正确起点（状态层）：下班前最后十分钟，工作消息还在响，但我知道我今天不会再回了——这是一种心理状态，用猫外化它。

对应拆解：

- 开头：消息还在响，但不反应
- 内部变化：从“我还在这里”，到“我不在这里”
- 凝固瞬间：盖上笔电
- 动作承载：每个动作都只是状态外化，不是为动作而动作

## 三｜评估标尺

### 3.1 主要标尺的倒置

| 本框架确立前 | 本框架确立后 |
|---|---|
| 画面精细度（主要） | 投射密度（主要） |
| 投射密度（次要 / 隐含） | 画面精细度（次要 / 辅助） |

### 3.2 投射密度评估维度

- 一级（定性）：目标用户能否在0.5秒内识别出“这是我的状态”？
- 一级补充：用户看完后的第一反应是“认同自己”，还是“评价猫”？
- 二级（发布后定量）：
  - 投射型评论占比（如“这就是我”“和我一模一样”）
  - 收藏率（是否愿意把它当作“自己状态的档案”）
  - 非算法驱动的二次传播率（是否主动转发给特定的人）

### 3.3 画面精细度的位置调整

画面精细度不被放弃，但降级为辅助标尺。

它的主要作用是：不阻碍投射。  
它的次要作用是：轻度加强投射（例如风格统一能让投射更稳定）。  
它不再是主要成功标准。

冲突时，以投射密度为准。

## 四｜下游动作推导

### 4.1 剧本设计

起点问题：这条视频服务什么心理状态？

操作：

- 先从赛道的心理状态清单里选一种具体状态
- 再按四步结构推导：开头 / 内部变化 / 凝固瞬间 / 动作承载
- 动作只作为状态外化设计，不再作为剧本起点

### 4.2 参考筛选

判断问题：这个参考在表达心理状态，还是只是在展示动作事件？

操作：

- 优先收集能触发目标用户投射的参考，典型信号是评论区高频出现“这就是我”“和我一模一样”
- 次选动作有趣、但投射不够强的参考
- 排除只有外表可爱、但没有投射价值的参考

### 4.3 视觉评估

第一问题不再是“像不像”“稳不稳”“精不精”，而是：它还能不能触发目标用户投射？

可接受：

- identity drift（只要还能触发投射）
- 节奏不均匀（只要核心瞬间清晰）
- 画面局部变形（只要不影响心理状态识别）

不可接受：

- 心理状态漂移（例如本来是“懒散豁免”，却变成“专注凝望”）
- 表情错误（例如紧张 / 警觉，导致投射对象改变）
- 场景错误（“那个我”本来不在这个空间里）

### 4.4 内容系列规划

推进逻辑不是“更多种猫的动作”，而是“更多种现代人的微心理时刻”。

每条视频 = 一种具体心理状态 × 一个适合外化它的动作片段。

系列化的增长方式，不是给猫增加越来越多设定，而是持续命名目标用户已经存在、但尚未被好好表达的状态。

### 4.5 文案撰写

文案的任务不是描述猫做了什么，而是为观众命名他们的心理状态。

- 错误：今天也是不想上班的小猫咪
- 正确：今天也想这样躺着
- 正确：谁还没有一个不想回消息的下午

### 4.6 数据指标体系

主要指标：

- 投射型评论占比
- 完播率 + 收藏率的组合表现
- 非算法驱动的二次传播

次要指标：

- 播放量
- 点赞数

## 五｜与其他核心文档的关系

### 5.1 与规划卡v1.0的关系

规划卡v1.0定义赛道在当期 sprint 中的具体执行维度（情绪内核、参考方向、猫适配方式、视觉分层、音频与文字层等）；本文档定义所有赛道背后的上游思维框架。

关系是：

- 本文档 = 上游战略
- 规划卡v1.0 = 下游执行

因此：

- 规划卡中的“情绪内核”必须从本文档的“心理状态”框架推导
- 规划卡中的评估维度，必须按“投射密度 > 画面精细度”的顺序调整

### 5.2 与 `06_宏观框架先行原则.md` 的关系

`06_宏观框架先行原则.md` 定义流程原则，回答“在进入执行层前，怎么准备”。

本文档定义内容原则，回答“到底要做什么样的内容，以及用什么标尺判断”。

两者不是替代关系，而是互补关系。每次 sprint 启动前，07 与 06 都必须先读。

### 5.3 与 `02_一条内容怎么做.md` / `03_什么样的内容算成立.md` 的关系

`02_一条内容怎么做.md` 定义最短生产闭环与角色位置，回答“流程怎么走”。  
`03_什么样的内容算成立.md` 定义视觉叙事规范与通过标准，回答“成品怎么看”。

本文档定义创作的战略锚点——所有判断的靶心是：投射是否成立。

因此：

- 02 负责把流程走对
- 03 负责把成品看对
- 07 负责把上游靶心定对

## 六｜使用方式

### 6.1 Sprint 启动时的默认顺序

1. 先读 `07_人的载体型剧本思维框架.md`
2. 再读 `06_宏观框架先行原则.md`
3. 基于当期赛道，明确“具体心理状态清单”
4. 按规划卡v1.0模板填写具体维度
5. 再进入参考搜集、作战卡、关键帧、视频生成与复盘

### 6.2 每次创作评估的默认顺序

先问：投射密度是否成立？  
再问：画面精细度是否足够？

- 一级不过关：回剧本层重做
- 一级过关 + 二级一般：允许继续进入后期
- 一级过关 + 二级也过关：进入闭环推进

### 6.3 冲突处理原则

当 prompt、画面、节奏、动作、场景、文案之间出现冲突时，统一回到核心问句：

这个决定，是服务投射密度，还是服务画面精细度？

前者优先，后者次要。
