# 09 变量耦合性与实验设计原则

## 核心摘要
- 实验设计的核心目的是：失败时能精确归因到具体变量。
- 教条式的"每次只改一个变量"不是真正的单变量控制——多个变量如果作用在不同元素上、视觉上可区分、不构成耦合，可以并行测试。
- 真正的判断标准是变量耦合性：是否在视觉/数据/逻辑上可独立观察。
- 不耦合的变量并行测试 → 高效率获得多个数据点。
- 耦合的变量必须分开测试 → 否则归因失败、实验白做。
- 本原则适用于视频生成实验、prompt实验、视觉风格实验等所有探索性工作。

## 零｜为什么需要这个文档

在赛道1的Seedance 2.0视频生成实验过程中，识别出策划者对"单变量控制"原则的理解过于教条：

- 错误理解：每次实验只改一个prompt词
- 正确理解：每次实验只改一个**耦合系统**

策划者最初的实验1设计想做5次实验（呼吸/眨眼/尾巴/窗帘/光斑各一次）——但实际上其中3个变量（呼吸/眨眼/尾巴）作用在猫身上是耦合的，应该分开测试；另外2个（窗帘/光斑）作用在不同元素上不耦合，可以并行。

这个洞察把5次实验缩减为2-3次实验，效率提升40%。本文档把这个原则一般化。

## 一｜核心命题

### 1.1 单变量控制的真正定义

不是"每次只改一个东西"——

而是"每次只改一个**耦合系统**"。

### 1.2 耦合性判断标准

两个变量是否耦合，看以下三个维度：

- **视觉可区分性**：变量A和变量B在画面上是否分别作用在可区分的元素上？
- **数据独立性**：变量A的成功/失败是否可以脱离变量B单独评估？
- **逻辑独立性**：变量A的逻辑是否依赖变量B的存在？

**全部三个维度都独立 → 不耦合 → 可并行**
**任一维度耦合 → 必须分开测试**

### 1.3 实战示例

**示例1｜视频生成实验**：
- 变量A：猫呼吸（作用在猫）
- 变量B：猫眨眼（作用在猫）
- 变量C：窗帘飘动（作用在窗帘）
- 变量D：光斑移动（作用在地面光斑）

A和B：耦合（都作用在猫身上，失败时无法区分是呼吸的问题还是眨眼的问题）→ 必须分开
C和D：不耦合（作用在不同元素，失败时一眼可见）→ 可并行
A+C+D：可并行（一次实验同时测3个数据点）
B单独：在A+C+D成功后，下一次实验加入B

**示例2｜prompt实验**：
- 变量A：色调描述（暖/冷）
- 变量B：构图描述（中心/偏移）
- 变量C：氛围描述（克制/戏剧）

A和C：耦合（色调和氛围在视觉感受上互相影响）
B：与A、C都不耦合（构图变化对色调/氛围影响弱）

可以一次实验同时测B+其中一个（A或C），但不能同时测A+C。

## 二｜实验设计三原则

### 原则1｜变量识别与耦合性判断

实验启动前必须明确：

- 本次实验要测哪些变量？
- 哪些变量两两之间耦合？
- 耦合矩阵建立后，识别出"独立变量集合"——可以并行测的最大组

### 原则2｜渐进叠加

成功的变量保留为下一次实验的"受控基准"——
失败的变量剔除，不带入下一次。

实验序列：
- 实验1：测试一组独立变量（X1, X2, X3）
- 实验2：基于实验1成功的基准，测试一个新变量（X4或X5）
- 实验3：基于实验2成功的基准，再加一个新变量
- ……

### 原则3｜归因可视化

每次实验失败必须能精确归因——

- 是哪个变量出了问题？
- 是变量之间的组合出了问题？
- 是变量之外的因素（工具默认行为/外部条件）出了问题？

如果失败时无法精确归因 → 实验设计本身有问题（违反耦合性判断）→ 重新设计实验。

## 三｜与市场参考的连接

变量的选择不是凭空——

每个变量的选择都应该基于市场已成功的范式（参考 `08_市场快照与预测性研究方法论.md` 和 `05_真实参考优先方法论.md`）。

如果某个变量的选择**找不到任何市场参考**——这个变量本身就是闭门造车——应该先验证再实验，而不是先实验。

## 四｜失败信号

- 实验失败但说不清楚是哪个变量的问题
- 一次实验测试了多个耦合变量
- 实验序列没有"基准+新变量"的结构
- 变量的选择没有市场参考支撑
- 把"变量"和"参数"混淆——参数（如时长/分辨率）是整体的，变量是元素层面的

## 五｜硬约束

1. 实验设计前必须完成耦合性判断。
2. 耦合变量必须分开测试，不允许一次性叠加。
3. 不耦合变量可以并行测试，但实验记录必须分开记录每个变量的表现。
4. 每次失败必须能归因——归因不出时停下来，不进入下一次实验。
5. 每次成功的基准必须明确记录——下一次实验在此基准上叠加新变量。
