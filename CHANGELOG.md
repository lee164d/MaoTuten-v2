# CHANGELOG

## 2026-04-24｜v2.0

### 触发
sprint赛道1的视频生成实验阶段，Lee识别出4个根本风险：
1. sprint前期所有上层工作（07原则、赛道、画像、美学）建立在没有市场数据底层支撑的假设上——可能是闭门造车
2. 策划者对"单变量控制"的理解过于教条——同元素的耦合变量被误用为"必须分开"，不同元素的独立变量被误用为"也必须分开"——效率折损
3. "抖音算法第一秒钩子"vs"猫图腾克制内核"的冲突需结构化解决方案
4. 60+次纠偏揭示策划者-决策者分工协议未被显式定义——项目可扩展性受限

由Lee提出并经过策略级确认：
- 在成熟生态里，爆款本身就是市场数据——直接看已成功者比看二级报告更直接
- 用预测对标而非当下对标，避开红海陷阱
- 套层策略让内核与流量算法结构化共存
- 决策者在ABC选项中选择，策划者列选项+评估+不抢跑

### 结论
- 新增 `08_市场快照与预测性研究方法论.md`：5步推演链作为所有方向决策的最上游
- 新增 `09_变量耦合性与实验设计原则.md`：实验设计的耦合性判断原则

## 2026-04-22｜v1.0

### 触发
sprint赛道1第一条视频在"前中后景首尾帧 → SD2直出 → Agent模式逐帧修复"迭代后，确认根本问题不在 prompt 或画质，而在上游定位没有被明确定义。

### 结论
- 猫图腾不是"漫改风格的猫IP（人格化内容形象）"，而是"人的心理状态的载体"。
- 创作起点从"猫要做什么动作"切换为"目标用户的什么心理状态要被外化"。
- 主要评估标尺从"画面精细度"切换为"投射密度"；画面精细度降级为辅助标尺。
- `07_人的载体型剧本思维框架.md` 与 `06_宏观框架先行原则.md` 并列，作为每次 sprint 启动前必读文件。

- 更新：`01_账号是什么.md`
- 待同步：规划卡v1.0模板（情绪内核来源改指向07；评估维度调整为"投射密度 > 画面精细度"）
