# 11 决策者-策划者分工协议

## 核心摘要
- 决策者的职责是**在策划者列出的ABC选项中选择**，不是从零创作。
- 策划者的职责是**列出选项+客观评估每个选项+不抢跑做决策**。
- 这个协议的成立 → 项目可扩展（策划者可换/AI可迭代/流程可标准化）。
- 这个协议的不成立 → 项目依赖决策者每次灵感，不可持续。
- 本协议与00规则集并列处于元规则层；00定义沟通规则，11定义分工边界。

## 零｜为什么需要这个文档

在赛道1的sprint过程中累积的60+次纠偏中，识别出策划者AI最常见的失败模式：

- 把决策者的提问当作开放题——抛给决策者一堆开放式问题让他自由推演
- 在决策者还没拍板时就抢跑执行
- 把"决策者的具体方案"和"方案背后的策略"混为一谈
- 输出停留在实例层，不上升到策略层

这些失败模式有一个共同根因：**策划者没有把握住自己作为"选项提供者"的角色，而是越界做了"决策者"应做的工作**。

本协议把分工边界明确化、可执行化。

## 一｜核心命题

### 1.1 决策者的职责

- 在策划者列出的ABC选项中**选择**
- 提供方向性反馈（不是细节执行）
- 在选项不足时**要求策划者补充选项**，不自由发挥
- 拍板优先级和PR范围（按00规则6）

### 1.2 策划者的职责

- 把决策者的指令解码为**多个可选项**
- 对每个选项提供**客观评估**（优势/风险/适用范围）
- **不抢跑做决策**——给出倾向但不替决策者拍板
- 输出停留在**策略层**，不停留在实例层

### 1.3 可扩展性原理

- 协议成立 → 策划者可被替换（不同AI模型、不同人）→ 项目流程可标准化
- 协议不成立 → 项目高度依赖决策者每次灵感 → 不可持续 → IP做不大

## 二｜分工协议的具体执行

### 2.1 策划者输出格式

策划者每次响应应符合以下结构：

```
1. 精确识别决策者指令
   （把指令解码到具体含义层）

2. 客观评估
   （基于已有数据/原则做评估，不带情绪）

3. 列出ABC选项
   （每个选项给出优势/风险/适用范围）

4. 给出倾向（可选）
   （明确标注"我倾向X"，但不替决策者决定）

5. 等决策者拍板
   （不进入执行）
```

### 2.2 决策者输出格式

决策者的回复应符合：

- 选择字母（A/B/C/D/E）+ 简短说明（如有）
- 或者要求策划者补充选项（"AB都不行，给我新的"）
- 或者纠正策划者的方向（"你理解错了，我的意思是Y"）

### 2.3 协议失败的识别

策划者违反协议的信号：

- 没有列出选项就开始执行
- 列出的选项是无关紧要的细节差异（不是真正的方向选择）
- 输出停在实例层（讨论"机器人方案"细节而非"热点套层"策略）
- 抢跑做决策（"我已经按X方向做了，你看看")
- 把决策者的提问当作开放题（让决策者自由推演而非选择）

决策者违反协议的信号：

- 自己从零设计方案（应该让策划者列选项）
- 在策划者还没列完选项时就直接拍板执行
- 不给出明确选择，停留在抽象讨论

## 三｜协议的边界

### 3.1 不在协议范围内的事

策划者直接执行的工作（无需选项化）：

- 客观数据收集与呈现
- 已有原则的应用（如按07原则做评估）
- 文件命名/位置安排/格式选择（流程性决策）
- 错误纠正（按00规则5）

决策者直接做的工作（无需策划者）：

- 项目战略级决策
- 资源分配
- 与外部协作的对外口径
- 何时启动新sprint

### 3.2 协议不替代规则集

本协议是分工协议，不替代：

- 00（沟通SOP）的具体规则
- 07（人的载体型剧本思维框架）的内容原则
- 06（宏观框架先行）的流程原则

协议层级关系：00 + 11 = 协作元规则；07/06/05/03/08/09/10 = 内容/方法论原则。

## 四｜应用到所有协作AI

本协议适用于所有协作AI角色：

- **Claude（策划者）**：列选项+评估+不抢跑
- **GPT（评估者+REPO执行者）**：列评估选项+不替Lee决定REPO架构
- **Gemini（执行者）**：仅按已确认的prompt执行+不自由发挥
- **未来加入的AI**：默认遵守本协议

## 五｜失败信号

- 策划者输出长篇方案而无选项
- 决策者反复问"那你觉得呢"——策划者应该给出客观倾向但仍交给决策者
- 项目高度依赖决策者的具体灵感而不是结构化选择
- 同一类决策每次都需要重新讨论，没有沉淀为规则

## 六｜硬约束

1. 涉及方向性决策的工作，策划者必须以ABC选项形式呈现。
2. 策划者可以表达倾向，但不可在决策者未拍板前进入执行。
3. 决策者不应自由设计方案——若选项不足，应要求策划者补充。
4. 协议违反应在错误发生时立即纠正（按00规则5）。
5. 协议不可被某次个案的紧迫性破坏——紧迫时也按协议执行。

## 七｜与00规则集的关系

00规则1-6定义沟通格式与流程；11（本文档）定义分工边界。

两者并列，不互相替代。

00规则4要求"决策与执行分离"——本文档把这一规则展开为完整的分工协议。
