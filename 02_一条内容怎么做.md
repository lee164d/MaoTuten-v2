用途：定义一条猫图腾内容从开始到完成的最短生产闭环，以及各角色在流程中的位置。
什么时候读：开始做任何一条视频前；角色越界或流程混乱时回看。
是不是唯一真源：是；关于最小生产闭环的定义以本文件为准。

# 02 一条内容怎么做

## 核心摘要
- 一条猫图腾内容不是"先想梗"，而是先锁定要承载的具体心理状态。
- 最短闭环只有6步：锁方向 → 找参考 → 写作战卡 → 出关键帧 → 生成视频 → 发布复盘。
- 默认顺序不是先写文字指令（prompt）再碰运气，而是先锁定真实参考结构，再让 AI 在该结构上生成变体。
- 角色不是独立世界观，而是流程里的功能位置。
- 角色边界由流程定义，不由角色文档自行扩张。

## 阶段 -1：宏观框架对齐（前置门）

进入本文档定义的任何执行步骤之前，必须先完成06文档定义的宏观框架对齐，产出"当期内容规划卡"。

规划卡的起点不是动作或梗，而是07文档定义的"这条内容要承载的心理状态"。先锁心理状态，再把该心理状态展开为宏观框架6个维度。

规划卡包含6个维度的方向性共识：赛道确认 → 情绪基调（对应心理状态） → 参考方向 → 猫适配方式 → 视觉分层方案 → 音频与语言层。

规划卡未产出，不进入下列任何阶段。

详见 `07_人的载体型剧本思维框架.md` 与 `06_宏观框架先行原则.md`。


## 最短生产闭环
### 0. 锁定情绪方向
- 先从07文档的心理状态清单出发，锁定本条内容要承载的具体心理状态。
- 再用热梗入口或对标账号入口验证：评论区最强共鸣是否指向同一心理状态。
- 只要猫IP（人格化内容形象）能作为这个心理状态的载体，就可以继续。
- 若只能找到"动作层共鸣"而找不到"心理状态层共鸣"，回07重新选心理状态。

### 1. 找参考源头
- 找到真正能作为视觉/节奏参考的源头内容。
- 参考源头优先使用真实社交平台中已被用户验证过的高赞猫类内容，不使用 AI 图或商业图库代替。
- 参考源头必须在"能外化该心理状态"上成立，不仅仅是"动作好看"。
- 参考源头必须由用户亲自确认。

### 2. 写作战卡
- 作战卡必须建立在已确认的真实参考结构上。
- 明确：这条视频要承载的心理状态是什么（而不仅是"交付什么动作"）。
- 明确：第1/2/3段结构（M1 / M2 / M3）分别是该心理状态的哪个内部阶段（状态 → 内部变化 → 凝固瞬间）。
- 明确：前景/中景/后景分别承担什么情绪功能，如何共同服务同一心理状态。
- 动作只是心理状态的外化载体，不是作战卡的出发点。

### 3. 出关键帧
- 先有能成立的关键静态画面（关键帧）。
- 关键帧成立的第一判断是投射密度，不是画面精细度。详见 `03_什么样的内容算成立.md`。
- 关键帧不成立，视频不成立。

### 4. 生成视频
- 让关键帧之间形成最小叙事弧线。
- 重点不是炫技，而是心理状态的内部变化是否连续成立。

### 5. 发布与复盘
- 数据不是终点，评论区反应才是认知升级入口。
- 复盘时首先回答：目标用户的反应是"这状态是我"还是"这只猫很可爱"。

## 角色在流程中的位置
- 用户：做最终判断，拍板继续 / 不继续（Go / No-Go）。
- 协调者AI（用户当前使用的对话界面）：在各角色之间传递信息，不做判断。
- 策划者 Claude：负责策划、拆解、单条视频结构说明卡（作战卡）、文字指令（prompt）、技术判断。
- 评估者 GPT：负责独立评估、交叉验证、仓库结构维护。
- 执行者 Gemini：只负责使用 Gemini 内部图像生成/编辑能力（NB2）进行图片生成与链式图像编辑。

## 角色边界硬约束
- 执行者 Gemini 不能自封策划者。
- 协调者AI不能替用户做判断。
- GPT 不负责图片生成。
- Claude 不替用户做最终美感裁决。

## 进入下一步的最低门槛
- 没锁定心理状态，不进下一步。
- 没有真实参考结构，不写作战卡。
- 没有真实参考结构，不进入纯文字 prompt 试错。
- 没有成立的关键帧，不进视频生成。
- 没有复盘，不算完成一次闭环。
