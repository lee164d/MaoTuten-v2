用途：定义猫图腾内容“成立 / 不成立”的判断标准，包括正向标准和负向约束。
什么时候读：写 prompt 前、看图时、评估通过与否时。
是不是唯一真源：是；关于内容是否成立的标准以本文件为准。

# 03 什么样的内容算成立

## 核心摘要
- 猫图腾内容成立，不看“画得多复杂”，先看“情绪是否交付”。
- 一级判断高于一切：目标用户第一秒是否能进入那种状态。
- 视觉叙事的最小结构是：前景张力 / 中景体态 / 后景节奏替换。
- 广告感 / 过度编排感默认视为失败信号，先回到真实参考结构检查，不优先靠 prompt 修辞补救。
- 对AI来说，不能画什么和必须画什么同等重要。

## 一级判断：情绪交付
- 核心问题：这张图 / 这条视频，能不能让目标用户在第一秒进入那个状态？
- 不看“比上次好不好”，只看“和目标方向是否对等成立”。
- 如果一级不通过，后面的构图、风格、参数都没有意义。

## 二级判断：三层情绪结构
### 前景
- 必须出现“本应关注但被搁置”的张力物件。

### 中景
- 猫的体态必须传达“什么都不做且不焦虑”。
- 重点不是表情丰富，而是体态卸力。

### 后景
- 必须用自然、缓慢、属于自己的节奏，替换社会催促节奏。

## 正向成立标准
- 第一秒能成立
- 猫仍然是猫，不是拟人角色皮套
- 情绪方向清晰，不是杂糅的概念拼贴
- 视觉元素各有情绪功能，不是为了好看硬堆
- 风格保持极简、克制、可识别

## 负向约束 / 禁止项
- 禁止猫穿全身人类服装
- 禁止做不像猫的动作
- 禁止把猫画成几乎认不出的拟人角色
- 禁止依赖微表情做情绪转折
- 禁止为了“精致感”牺牲情绪命中
- 禁止堆过多独立小物件制造伪丰富
- 禁止低俗、吵、乱、土
- 禁止没有真实参考结构支撑就凭纯文字想象构图
- 禁止出现明显广告式开场 / 过度编排感画面

## 通过的最低条件
- 用户看完不会说“结构不错但像个未完成样稿（demo）”
- 如果画面第一眼像商业图 / 样片 / 摆拍图，直接不通过
- 目标用户不是只看到“猫可爱”，而是会感到“这状态是我”
- 画面移除任一关键层后，情绪明显受损
