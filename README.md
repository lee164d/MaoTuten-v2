# 猫图腾｜MaoTuten

猫图腾不是宠物博主账号，而是一个用银白猫IP（人格化内容形象）作为城市独居女性"数字替身"的短视频账号，目标平台是小红书 / 抖音。

## 为什么存在

它存在的目的，不是展示猫有多可爱，也不是展示AI画技，而是把评论区里真实存在的情绪方向，转译成用户可代入的猫IP内容。用户要的不是猫——而是"这只猫替我活出了我想要的状态"。

## 它怎么运转

先锁定要承载的心理状态，再锁定情绪方向与参考结构，写单条视频结构说明卡（作战卡），出关键静态画面（关键帧），生成视频，最后发布复盘。它不是"先想一个梗再包装"，而是先确认：这只猫能不能替目标用户活出那个心理状态。

## 什么叫成立

成立不看"画得多复杂"，先看投射密度——目标用户第一秒能否认出"这状态是我"。画面精细度是辅助标尺，不能替代投射密度。同时猫必须仍然像猫，而不是拟人角色皮套；画面里的前景 / 中景 / 后景必须共同承担同一心理状态的外化功能。

## 当前在做什么

当前主方向是赛道1（当前正在验证的第一条情绪赛道）「独立自足型」。现在的问题不是缺规则，而是视觉完成度和情绪浓度还没到成品级，所以仍在继续跑场景测试画面（场景帧）测试。

## 先读顺序

注：`00_协作者沟通SOP.md` 是所有AI协作者的常驻行为规则，不在上述按顺序阅读的主流程中，但所有协作者在接入项目时必须首先阅读。

`07_人的载体型剧本思维框架.md` 与 `06_宏观框架先行原则.md` 并列处于最上游，两者都必须先于其他文档被阅读：
- 07 负责回答"这条内容承载什么心理状态"。
- 06 负责回答"怎么把这个心理状态展开成可执行的宏观框架"。

读完 07 + 06 后，按以下顺序继续阅读：

1. `01_账号是什么.md` — 项目定义、猫IP人格核、目标受众
2. `02_一条内容怎么做.md` — 阶段0-6生产闭环、决策节点、角色分工
3. `05_真实参考优先方法论.md` — 真实参考结构优先于纯文字 prompt 的默认工作顺序与硬约束
4. `03_什么样的内容算成立.md` — 视觉叙事规范、三层情绪结构、投射密度优先的通过标准
5. `04_现在在验证什么.md` — 当前状态、待验证事项、对标账号、工具事实
6. `tools/NB2_避坑指南.md` — Gemini 内部图像生成/编辑能力（NB2）已知限制与避坑清单
