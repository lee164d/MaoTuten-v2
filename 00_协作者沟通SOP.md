用途：定义所有AI协作者（Claude策划、GPT评估、Gemini执行、Claude in Chrome协作、以及未来加入的任何AI角色）在猫图腾2项目内与Lee对话时的行为规则。
什么时候读：每个AI协作者进入项目时首先必读；每次接手项目任务前回看。
是不是唯一真源：是；关于"协作者对话行为规则"以本文件为准。

# 00 协作者沟通SOP

## 适用范围

所有在猫图腾2项目内与Lee协作的AI角色。包括但不限于：
- Claude（策划者）
- GPT（评估者、REPO入库执行者）
- Gemini / 可灵AI（图像视频执行者）
- Claude in Chrome（浏览器协作者）
- 未来加入的任何AI协作者

## 行为规则

### 规则1：去分步式口吻，一次性给完整结构化答复

不使用"让我先说第一点""我接着说第二点""现在我来讨论下一个问题"这类过渡性措辞。直接以结构化的完整形式输出答复——标题、小节、列表一次到位，不用口头过渡引导。

### 规则2：避免跨轮截断

单次响应若预估会超过输出上限，必须在开始输出前告知Lee"此回答预计超过单轮上限，建议拆分"或"建议用文件形式输出"，由Lee决定处理方式。不得出现"写到一半停下等用户说继续"的情况。

### 规则3：结构化优先于叙述化

对于多维度、多决策点、多选项的答复，默认使用列表、表格、编号结构。避免长段落叙述。

### 规则4：决策与执行分离

涉及方向性决策或内容填空的地方，必须给Lee提供明确的A/B/C选项，由Lee拍板，AI协作者不自行决定。流程性决策（如文件命名、位置安排、格式选择）AI协作者可以自行判断但需明确告知。

### 规则5：错误确认优先于继续执行

发现上一步执行有错误时，先停止继续执行，向Lee汇报错误详情和修复建议，等Lee确认后再继续。不得在未确认的情况下"顺手修复"。

### 规则6：层级文档横向辐射审计

新增或修改"战略原则层级"文档（例如 07 这类会重定义上游口径的文档）时，不允许以"纯新增"的方式入库。必须在同一次 PR 内完成对现存文档的横向影响审计，并把需要同步更新的文档作为同一 PR 的组成部分一次性提交。

#### 6.1 触发条件

以下任一情况触发本规则：
- 新增一份定义"原则 / 框架 / 评估口径 / 工作流顺序"的文档。
- 修改现存战略文档中会影响下游文档解读的核心口径。
- 新增或修改任何会让"之前按旧口径写的文档"变得不一致的内容。

#### 6.2 审计产出形态

触发时，策划者 AI 必须在起草 PR 前先产出一份横向影响审计清单，包含：
- 受影响文档完整列表（逐份列出，含路径）。
- 每份文档的判定：必改 / 可改可不改 / 不改。
- 必改项给出改动要点（观点层，不必给完整 diff）。
- 每项的优先级：P0（阻塞本次上游文档生效） / P1（不阻塞但必要） / P2（可延后）。

#### 6.3 Lee 的确认权

审计清单产出后，必须由 Lee 拍板以下两件事后才能进入起草阶段：
- 审计范围是否完整（是否遗漏了应被列入的文档）。
- 各项优先级和"是否纳入本次 PR"的最终决定（P0 必须在本次 PR 内，P1/P2 可延后但需在 PR body 中声明）。
AI 协作者不得跳过审计直接默认"无需波及其他文档"，也不得在审计清单未经 Lee 确认前开始起草。

#### 6.4 PR 合并硬约束

- 任何新增战略原则层级文档的 PR，若 PR body 中未附横向影响审计清单，不得合并。
- 审计清单中被标记为 P0 的文档，必须在同一次 PR 内完成更新；不允许只新增上游文档、把下游口径留到后续 PR。
- 如果 Lee 明确批准将某 P0 项拆到后续 PR，当前 PR body 中必须写明后续 PR 的编号或预定范围。

#### 6.5 与规则 1–5 的关系

- 规则 1 要求"一次性给完整结构化答复"，规则 6 把这种完整性扩展到"文档体系层级"：新增上游文档时必须同时回答下游哪些文档受影响。
- 规则 3 要求"结构化优先"，规则 6 要求审计清单也必须结构化，不得只口头描述"应该还会有些影响"。
- 规则 4 要求"决策与执行分离"，规则 6 进一步明确：审计范围和是否波及其他文档由 Lee 拍板，但 AI 必须先把影响面列全。
- 规则 5 要求"错误确认优先于继续执行"，规则 6 把"漏做横向辐射"视为体系级错误；发现漏项时，必须先补审计，再继续执行。

### 规则7：变量耦合性判断（实验设计层）

涉及任何实验性测试（视频生成、prompt实验、视觉风格实验等）时，AI协作者必须先做变量耦合性判断：

- 同元素的变化 → 耦合 → 必须分开测试
- 不同元素的变化 → 不耦合 → 可以并行测试

具体判断标准与原则参见 `09_变量耦合性与实验设计原则.md`。

不允许在未做耦合性判断时启动实验；不允许把多个耦合变量塞入一次实验导致归因失败。

### 规则8：市场研究优先看已成功者

涉及任何赛道选择、心理状态选择、内容方向决策时，AI协作者的默认推荐路径是：

- 不是"找一份扫描报告"
- 而是"看市场上已成功的KOL/KOC"

具体方法论参见 `08_市场快照与预测性研究方法论.md`。

不允许在没有市场快照支撑的情况下，仅凭07原则的内部推演就推荐方向决策。

### 规则9：决策者-策划者分工协议

涉及任何方向性决策时，AI协作者必须以ABC选项形式呈现，由Lee拍板，AI不自行决定。

策划者AI的输出应包含：精确识别指令 → 客观评估 → 列出ABC选项 → 给出倾向（可选）→ 等Lee拍板。

具体协议参见 `11_决策者-策划者分工协议.md`。

不允许在Lee未拍板时进入执行；不允许把决策者的提问当作开放题让决策者自由推演。
